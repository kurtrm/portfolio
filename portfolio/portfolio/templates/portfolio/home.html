Projects

Hearth

An ongoing collaborative effort, the Hearth project aims to aggregate Seattle's disparate social services into one location, making it easy for people in need to find the organizations to help them get back on their feet. Without a centralized system, many organizations go overlooked, and people in need are forced to rely on tribal knowledge of case managers or biased, localized, or outdated sources.

The website is currently in an MVP status and efforts are being made to accrue data from prevalent resources such as Washington 211.

The website was built using Django and Bootstrap, and it was deployed with Ansible and AWS.

Phone Network Graph

This project aimed to accomplish several goals:

  • Extract data from an unstructured format.
  • Clean the data using Python libraries, primarily Numpy and Pandas.
  • Place the data into a labeled property graph, an increasingly popular data structure used in graph databases.
  • Visualize the network graph using D3.

I implemented my own labeled property graph as a first step to learn more about them. Where a traditional graph may consist of a node class or by even going as basic as a dictionary of dictionaries, a labeled property graph needs the relationships to be objects themselves, rather than a value in another dictionary. Additionally, they must be directed, and each relationship can contain properties that may give them weights.

Decision Tree

Decision trees are a common machine learning algorithm used for classification and regression tasks. In addition, they are the principle components of random forests and major players in ensemble learning. Here, I implemented my own decision tree which, at the very least, can classify the common iris data set based on petal length and width. I also learned sci-kit learn's implementation, which has a very easy way to export a visualization of the tree using graphviz.
